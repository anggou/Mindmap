## 코딩 규칙

1. 코드는 **Python**을 사용한다.
2. UI 프레임워크는 **Streamlit**을 사용한다.
3. 코드 생성 이후 반드시 다음 절차를 수행한다:
   - 생성된 코드를 **실행하여 정상 동작 여부를 확인**한다.
   - 실행 중 **버그(에러)가 발생하면 자동으로 디버그 절차를 수행**한다.
     - 에러 메시지를 분석하고 원인을 파악한다.
     - 코드를 수정한 뒤 다시 실행하여 검증한다.
     - 모든 에러가 해결될 때까지 이 과정을 반복한다.
   - **정상 실행이 확인된 후에만** 작업을 종료한다.

## API 연동 규칙

1. AI 모델 API 키는 반드시 **`.env` 파일** 또는 **Streamlit 사이드바 입력**으로 관리한다.
2. `.env` 파일은 절대 Git에 커밋하지 않는다 (`.gitignore`에 포함 필수).
3. 외부 API 호출(Gemini, OpenAI, Anthropic)은 반드시 `try-except`로 감싼다.
4. API 호출 실패 시 사용자에게 `st.error()`로 명확한 에러 메시지를 표시한다.

## 프로젝트 구조 규칙

1. Streamlit 앱 진입점은 `app.py` 단일 파일로 유지한다.
2. 지원 AI 모델: Gemini (Google), GPT (OpenAI), Claude (Anthropic).
3. 다국어 지원: 한국어/English 번역 테이블(`TRANSLATIONS` dict)로 관리한다.
4. 모든 UI 텍스트는 번역 테이블 `T["key"]`를 통해 참조한다 (하드코딩 금지).
5. 환경 변수 샘플은 `.env.example`에 문서화한다.

## Mermaid 마인드맵 규칙

1. AI에게 Mermaid 코드 생성을 요청할 때 반드시 `mindmap` 형식을 지정한다.
2. 루트 노드는 `root((주제))` 형식, 최대 3단계 깊이, 노드 텍스트 20자 이내.
3. 렌더링은 **mermaid.js CDN** (v11)을 통해 브라우저에서 수행한다.
4. AI API 응답 파싱: `## 핵심 요약` + `## Mermaid 마인드맵` 구간을 정규식으로 분리한다.
